=========================================
 GUIA DE EXECUÇÃO - GOOGOL (Meta 2)
=========================================

PASSO 0: PREPARAÇÃO (Executar uma vez)
-----------------------------------------
1. Iniciar o serviço de IA (em background):
   ollama serve

2. Compilar o projeto:
   mvn clean compile

=========================================
PASSO 1: ARRANCAR SISTEMA (Terminais Separados)
Nota: Abre um novo terminal para cada comando abaixo.
=========================================

# Terminal 1 - Barrel 0 (Storage)
java -cp target/classes pt.uc.sd.googol.barrel.BarrelLauncher 0

# Terminal 2 - Barrel 1 (Storage Backup)
java -cp target/classes pt.uc.sd.googol.barrel.BarrelLauncher 1

# Terminal 3 - URL Queue (Fila de Tarefas)
java -cp target/classes pt.uc.sd.googol.queue.QueueServer

# Terminal 4 - Gateway (Servidor RMI Principal)
java -cp target/classes pt.uc.sd.googol.gateway.Gateway

# Terminal 5 - Downloader (Web Crawler)
mvn exec:java -Dexec.mainClass="pt.uc.sd.googol.downloader.Downloader"

=========================================
PASSO 2: INTERFACES DE UTILIZADOR
=========================================

# Terminal 6 - Web Server (Frontend Meta 2)

mvn spring-boot:run

Nota sobre a Ligação Segura Para simular um ambiente de produção real,
o projeto corre sobre HTTPS na porta 8443. Como estamos a usar um certificado
auto-assinado para fins académicos, o browser irá apresentar um aviso de
"Sítio Não Seguro" ou "A sua ligação não é privada". Deverá ignorar este
aviso e forçar a entrada (Geralmente em: Avançadas -> Continuar para...)
para que a comunicação WebSocket funcione corretamente.

-> Aceder em: https://10.46.221.165:8443

# Terminal 7 - Cliente CLI (Opcional - Meta 1)
java -cp target/classes pt.uc.sd.googol.client.GoogolClient
